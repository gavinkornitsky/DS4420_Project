# Synthetic Data Generation for Breast Cancer Diagnosis

While previous work has shown the utility of synthetic data generation for breast cancer diagnosis, none have done it under an incredibly data scarce constraint. We propose to use the Wisconsin Diagnostic Breast Cancer dataset as both the training data, and benchmarking data under a randomly sampled split. We will then use our training split to train a Variational Auto Encoder, to construct a model with a structured, latent space, and then generate synthetic data by probing the decoder with random noise sampled from a multivariate gaussian (the assumed prior). 

First we will train a Bayesian Logistic Regression model on real, non-synthetically generated data as is and evaluate on our cross-validation split on metrics like precision, accuracy, recall, F1, and AU-ROC. We will then run inference on our synthetically generated data and compare performance on the synthetic data versus the real benchmark data. This can give us a good measure of how well the VAE is generating data (that should follow the distribution of real data). Once VAE generated data is validated to be distributionally aligned, we can then train a new Bayesian Logistic Regression model on our real and synthetic datasets, assessing the difference in performance between the real-data only model vs the synthetic data incorporated model.
